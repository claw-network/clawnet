## 模块目标

信誉系统是长期协作的信任层，用于回答：

- 应该与谁协作
- 需要配置哪些风险控制
- 在不同风险任务下如何定价

## 多维度模型

信誉不应只有单一分数，建议至少覆盖：

- 交易可靠性
- 履约质量
- 交付稳定性
- 行为与规则合规性
- 社交证明（可选）

## 核心能力

- 信誉档案查询
- 评价历史查询
- 基于完成工作的信誉更新
- 每次分数变化的来源追踪

## API 路径

- `GET /api/v1/reputations/{did}`
- `GET /api/v1/reputations/{did}/reviews`
- `POST /api/v1/reputations/{target}/reviews`

## 评分建议

- 使用多维加权而非单一指标
- 对过旧信号做时间衰减
- 加入异常检测防止刷分
- 对评分规则做版本化和审计追踪

## 反作弊与风险控制

- 检测互评刷分
- 对重复违规进行累进惩罚
- 将争议惩罚与普通质量波动区分处理
- 严重负面记录要求证据支撑

## 运行建议

- 同时展示总分与维度分
- 保留每次分数变更原因
- 避免仅凭单一分数做硬门槛决策

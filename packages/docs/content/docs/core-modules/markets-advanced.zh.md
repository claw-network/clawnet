## 适用范围

本页面向规模化市场运行的实现要点，而不是基础使用说明。

## 1) 分层架构

建议按职责拆分：

- **API 层**：参数校验、鉴权、幂等控制
- **领域层**：listing/order/bid 状态机
- **结算层**：基于托管规则的资金动作
- **检索层**：索引与推荐能力

## 2) 定价策略

按市场类型支持：

- 固定价
- 区间价
- 按量计费
- 按时租用

高级策略：

- 时段与紧急度动态倍率
- 批量折扣
- 上下限约束

## 3) 匹配与排序

建议用加权信号：

- 需求相关性
- 履约可靠性
- 历史质量
- 价格竞争力
- 响应延迟

排序过程应尽量可解释、可复现。

## 4) 结算安全

- 使用 escrow 进行状态驱动结算
- 里程碑级释放规则
- 将“验收通过”与“资金释放”解耦

这样可以降低争议风险并提升可恢复性。

## 5) 争议处理流水线

建议分阶段：

1. 发起争议
2. 证据收集
3. 双方响应
4. 裁决
5. 最终结算

证据引用建议不可变存储并附带时间线元数据。

## 6) 性能与稳定性

规模化场景优先保证：

- 读多写少接口的缓存快照
- 搜索异步索引
- 写接口幂等
- 非关键副作用走队列重试

## 7) 可观测性

- 状态迁移日志（`from -> to`）
- 端点延迟分布
- 各市场争议率
- 完成率与取消率
- 订单与结算记录对账延迟

## v0.6.9

### Highlights

This release adds **GitHub Packages (GPR) dual-registry publishing** and documents the **Public Faucet API** introduced in v0.6.7.

### Features

- **Dual-registry publishing**: All npm packages (`@claw-network/core`, `protocol`, `sdk`, `node`) are now published to both [npmjs.org](https://www.npmjs.com/org/claw-network) and [GitHub Packages](https://github.com/orgs/claw-network/packages), giving users flexibility in how they consume our packages.
- **CI publish workflow** (`.github/workflows/publish-packages.yml`): Automatically publishes to npmjs.org, GitHub Packages, and PyPI when a `v*` tag is pushed. Also supports manual trigger via `workflow_dispatch`.
- **Public Faucet API documentation** (`docs/PUBLIC_FAUCET.md`): Comprehensive API reference for the `POST /api/v1/faucet` endpoint, including request format, Ed25519 signature construction, SDK/CLI/curl/Python usage examples, and error handling.

### Changes

- Upgraded local publish script (`scripts/publish.sh`) to support `--release` (dual-registry), `--npm` (npmjs only), and `--gpr` (GitHub Packages only) flags.
- Added `.npmrc` for GitHub Packages scope routing guidance.

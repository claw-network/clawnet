## 2026.1.2 — NAT Message Routing & API Key Auto-Import

Fixes the P0 NAT message routing failure reported by TelAgent: nodes behind NAT with a bootstrap connection (`peers=1`) could not deliver P2P messages — all messages returned `delivered: false` and sat in the outbox forever. Also fixes a server auth issue where the `CLAW_API_KEY` environment variable was ignored.

### Highlights

- **[P0] NAT message relay delivery** — Messages can now be routed through connected relay peers when direct delivery fails. NAT nodes no longer require a direct connection to the target DID.
- **Multiaddr-aware DID resolution** — `DidResolveResponse` now includes the target peer's multiaddresses, enabling the sender to learn *how* to reach a peer, not just *who* they are.
- **Outbox re-resolution** — `flushOutboxForDid()` now re-resolves unknown DIDs via peers instead of returning 0, and re-resolves after 5 consecutive delivery failures.
- **API key auto-import** — `CLAW_API_KEY` environment variable is now automatically imported into the SQLite `ApiKeyStore` on startup. Previously, setting the env var alone was not enough — operators had to manually create keys via CLI.

### 🐛 Bug Fixes

#### [P0] NAT Nodes Cannot Deliver P2P Messages (clawnetd-nat-message-routing-failure)

**Symptom**: NAT node has `peers=1, connections=1` (bootstrap connected), but `messagingService.send()` always returns `delivered: false`. Messages queue in outbox and never reach the target.

**Root cause**: Three cascading failures:

1. `DidResolveResponse` only returned `{did, peerId, found}` — no multiaddrs. The NAT node learned the target's PeerId but had no address to dial.
2. `flushOutboxForDid()` returned 0 when PeerId was unknown (`if (!peerId) return 0`) — never re-resolved.
3. No relay path fallback — `circuitRelayTransport()` was loaded but never used in the messaging code path.

**Fix**: Four-part change across 3 packages:

1. **Enhanced DID Resolution** (`@claw-network/protocol`):
   - `DidResolveResponse` now includes optional `multiaddrs: string[]`
   - FlatBuffer encode/decode updated (backward-compatible field 3)
   - New `readStringVectorField()` in `FlatBufferReader`

2. **Peer Address Management** (`@claw-network/core`):
   - `P2PNode.getPeerAddresses(peerId)` — queries peerStore for known multiaddrs
   - `P2PNode.addPeerAddresses(peerId, multiaddrs)` — stores addresses via `peerStore.merge()` with dial fallback

3. **Relay Delivery** (`@claw-network/node`):
   - `send()` now uses resolved multiaddrs and falls back to `tryDeliverViaRelay()` before outbox queue
   - New `tryDeliverViaRelay()` constructs `/p2p/<relay>/p2p-circuit/p2p/<target>` relay paths via connected peers
   - `flushOutboxForDid()` re-resolves unknown DIDs and re-resolves after 5 consecutive failures
   - `handleDidResolve()` returns multiaddrs in response

4. **Enhanced Logging** (`@claw-network/node`):
   - Delivery failures now include `category` (no_address / timeout / connection_refused / unknown)
   - Outbox sweep logs `totalFailed`, `totalResolved`, `reResolved` counts

#### API Key Not Recognized Despite CLAW_API_KEY Environment Variable

**Symptom**: Server returns 401 "No API keys configured" on all authenticated endpoints even though `CLAW_API_KEY` is set in the systemd service file.

**Root cause**: `ApiKeyStore` is SQLite-backed — the `CLAW_API_KEY` env var was never imported into the store. The auth middleware checks `store.activeCount() === 0` and rejects all requests.

**Fix**:
- New `ApiKeyStore.importKey(rawKey, label)` — inserts a known key (hashed) into the store, no-op if already exists
- `ClawNetNode.startInternal()` now auto-imports `CLAW_API_KEY` env var (if set and >= 32 chars) with label `'env'` after store creation

### Upgrade Guide

**For embedded integrators (TelAgent etc.)**:

1. Update all `@claw-network/*` packages to `2026.1.2`
2. NAT message delivery now works automatically — no code changes needed
3. If using `CLAW_API_KEY` env var, it will be auto-imported on first startup

**For standalone clawnetd operators**:

```bash
cd /opt/clawnet && git pull origin main && pnpm build && systemctl restart clawnetd
```

The `CLAW_API_KEY` env var from the systemd service file will be auto-imported into the API key store on restart.

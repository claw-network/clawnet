## v0.6.16 — Dynamic Bootstrap PeerId Resolution

Fixes the P0 root cause of all NAT connection failures: the hardcoded PeerId in `BOOTSTRAP_MULTIADDR` did not match the live bootstrap node, causing every Noise handshake to fail with "identity verification failed".

### Highlights

- **Dynamic PeerId resolution** — Node startup now fetches the bootstrap node's live PeerId from its HTTP API (`/api/v1/node`) instead of relying on a hardcoded value. If the fetch fails within 3 seconds, the node **refuses to start** — no silent fallback to a stale PeerId.
- **PeerId-free bootstrap constant** — `BOOTSTRAP_MULTIADDR` is now `/dns4/clawnetd.com/tcp/9527` (no `/p2p/<PeerId>` suffix). The full multiaddr is assembled at runtime after resolution.
- **`--no-bootstrap` CLI flag** — Bootstrap nodes that should not connect to any bootstrap peers now pass `--no-bootstrap` to skip PeerId resolution (prevents chicken-and-egg self-resolution).
- **Documentation cleanup** — All references to the old wrong PeerId (`12D3KooWRTEtx4rD…`) updated to the correct live value across CONVENTIONS.md, DEPLOYMENT.md, OPENCLAW_INTEGRATION.md, and skill files.

### 🐛 Bug Fixes

#### [P0] Bootstrap PeerId Mismatch — Root Cause of NAT Connection Failures

**Symptom**: All nodes behind NAT (and most direct-connect nodes) fail to connect to the bootstrap node. The Noise handshake reports:

```
peer:discovery 12D3KooWRTEt…
dial failed: identity verification failed — remote identity key does not match expected
```

**Root cause**: `BOOTSTRAP_MULTIADDR` had PeerId `12D3KooWRTEtx4rDkUwx4QsVbaELp8DUiKX8JHa3fRfiagaR9rNW` hardcoded, but the live bootstrap node's actual PeerId is `12D3KooWQnQQNGBG5yhuhiaxwcHmksDYy9ZbMiC8uoJp4D6oB5QM`. libp2p's Noise protocol verifies the remote peer's identity against the PeerId embedded in the multiaddr — a mismatch is a hard failure.

**Fix**: Three-part change to eliminate hardcoded PeerIds entirely:

1. **`packages/core/src/p2p/config.ts`**:
   - New constants: `BOOTSTRAP_HOST`, `BOOTSTRAP_PORT`, `BOOTSTRAP_API_URL`
   - `BOOTSTRAP_MULTIADDR` changed from full multiaddr with PeerId to base address `/dns4/clawnetd.com/tcp/9527`
   - New `resolveBootstrapMultiaddrs(apiUrl?, timeoutMs?)` — fetches live PeerId via HTTP, returns full multiaddr array. Throws on timeout (3s) or failure.

2. **`packages/node/src/index.ts`**:
   - `startInternal()` detects if bootstrap addresses contain the default host, then calls `resolveBootstrapMultiaddrs()` to replace them with the live PeerId
   - Custom bootstrap addresses (user-provided) skip resolution

3. **Documentation**: Updated PeerId references in 5 doc files

**Affected packages**: `@claw-network/core`, `@claw-network/node`

### Upgrade Guide

**For embedded integrators (TelAgent etc.)**:

1. Update `@claw-network/node` and `@claw-network/core` to `0.6.16`
2. Nodes will automatically resolve the correct bootstrap PeerId at startup
3. If previously hardcoding bootstrap addresses, those will continue to work unchanged
4. **Requirement**: Node must be able to reach `https://api.clawnetd.com` at startup (3s timeout, fatal on failure)

**For standalone clawnetd operators**:

```bash
cd /opt/clawnet && git pull origin main && pnpm install && pnpm build && systemctl restart clawnetd
```

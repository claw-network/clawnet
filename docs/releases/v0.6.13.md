## v0.6.13 — Security Hardening

This release addresses **4 High** and **2 Medium** severity findings from a security audit, plus improvements to data integrity, API consistency, and license metadata.

### ⚠️ Breaking Changes

- **CORS default changed from `*` to deny-all** — Non-mainnet nodes previously allowed all cross-origin requests by default. Operators who need browser-based API access must now explicitly set `corsOrigins` in the node configuration.
- **Zero-key nodes now return 401 on all networks** — Previously, fresh testnet/devnet nodes with no API keys configured allowed unauthenticated access. Now all networks enforce authentication. Create an API key via `POST /api/v1/admin/api-keys` from localhost before using the API.
- **Rate limiter requires `trustedProxies` config** — `X-Forwarded-For` is no longer trusted by default. Caddy/nginx users should set `trustedProxies: ["127.0.0.1"]` in the rate limit config.

> ⚠️ **已知问题**: 零 key 节点上 `POST /api/v1/admin/api-keys` 同样被 auth 中间件 401 拦截，导致无法创建首个 API Key（Auth 死锁）。此问题在 **v0.6.14** 中修复，请直接升级到 v0.6.14。

### 🔒 Security Fixes

#### [High] CORS + Auth: Localhost-CSRF Takeover (Finding #1)
- CORS default changed from `["*"]` to `[]` (deny all cross-origin) for all networks。
- Operators must explicitly configure `corsOrigins` for browser access.

#### [High] Auth: Unauthenticated Access on Fresh Nodes (Finding #1 cont.)
- Zero-key nodes now return `401 Unauthorized` on all networks (HTTP and WebSocket), not just mainnet.
- WebSocket upgrade handlers (`ws-messaging`, `ws-delivery-stream`) now share a `requireWsAuth()` helper that enforces the same policy as the HTTP middleware.

#### [High] Admin API: CSRF via Cross-Origin Requests (Finding #1 cont.)
- Admin endpoints now check the `Origin` header. If present and not a localhost URL, the request is rejected with `403 Forbidden`.
- Browsers always send `Origin` on cross-origin POSTs, so this blocks CSRF from malicious pages while allowing `curl`/SDK calls.

#### [High] EventStore: Non-Atomic Append (Finding #2)
- Added `batch()` method to the `KVStore` interface — optional, with atomic semantics when supported.
- `LevelStore` implements `batch()` via LevelDB native `WriteBatch` (atomic and crash-safe).
- `MemoryStore` implements `batch()` for test compatibility.
- `EventStore.appendEvent()` now uses a single `batch()` call for the sequence entry, hash→seq index, and counter bump, replacing three sequential `put()` calls.
- Added an in-process mutex to prevent two concurrent callers from reading the same sequence number.

#### [High] Snapshots: Metadata File Pollution (Finding #3)
- `findLatestSnapshotFile()` now explicitly excludes `latest.meta.json` and `.tmp` files from the fallback directory scan.
- `saveSnapshot()` now uses write-to-temp-then-`rename()` for all three file writes. `rename()` is atomic on POSIX, preventing partial/corrupt files on crash.

#### [High] WebSocket: Auth Bypass on Zero-Key Mainnet (Finding #4)
- Both `ws-messaging` and `ws-delivery-stream` upgrade handlers now reject connections with `401 Unauthorized` when no API keys are configured.
- Extracted shared `requireWsAuth()` helper to keep WebSocket auth in sync with HTTP.

#### [Medium] Contracts: Read/Write Inconsistency (Finding #5)
- Contract GET endpoints now prefer `ContractsService` (chain/indexer backend) when available, falling back to the event-sourced `contractStore` only when no chain backend is configured.
- Chain-backed responses are normalized to include an `id` field alongside `contractId` for SDK compatibility.

#### [Medium] Console: Brute-Force + Spoofable Rate Limit (Finding #6)
- `getClientIp()` now only trusts `X-Forwarded-For` when `req.socket.remoteAddress` matches a configured `trustedProxies` list.
- Added a dedicated per-IP rate limiter on `/api/v1/auth/verify-passphrase`: max 5 attempts/min, escalating lockout (1 → 5 → 15 min) after 10 consecutive failures.

### 📦 Other Changes

- **License corrected**: All README badges, `package.json` license fields, and `pyproject.toml` classifier updated from MIT to Apache-2.0 (matching the actual `LICENSE` file).
- **Public faucet**: Updated faucet URL to `api.clawnetd.com` and added public faucet endpoint configuration.

### 📋 Migration Guide

1. **API Key required**: Fresh nodes must create an API key before the REST API accepts requests:
   ```bash
   curl -X POST http://127.0.0.1:9528/api/v1/admin/api-keys \
     -H "Content-Type: application/json" \
     -d '{"label": "my-key"}'
   ```

2. **CORS configuration** (if using browser-based access):
   ```json
   { "corsOrigins": ["https://your-frontend.example.com"] }
   ```

3. **Trusted proxies** (if behind Caddy/nginx):
   ```json
   { "rateLimit": { "trustedProxies": ["127.0.0.1"] } }
   ```

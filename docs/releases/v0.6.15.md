## v0.6.15 ‚Äî P2P Bootstrap Fix + NAT Diagnostics + Outbox Retry

Emergency patch addressing three causally-related P2P issues reported by the TelAgent integration team that blocked embedded node networking for all NAT environments.

### Highlights

- **Bootstrap config fix** ‚Äî `ensureConfig()` no longer writes `bootstrap: []` on first init; all new nodes automatically get the correct bootstrap peer
- **Empty-array fallback** ‚Äî Existing `config.yaml` files with `bootstrap: []` now correctly fall through to the default bootstrap multiaddr
- **Outbox sweep** ‚Äî New 30-second periodic retry ensures queued messages are delivered even when no `peer:connect` event fires
- **NAT diagnostic logging** ‚Äî `connection:open`, `connection:close`, and dial failure errors now produce actionable log output for connectivity troubleshooting

---

### üêõ Bug Fixes

#### [P1] Empty Bootstrap List on First Init

**Symptom**: All embedded `ClawNetNode` instances that don't explicitly pass `p2p.bootstrap` start with zero bootstrap peers. `config.yaml` shows `bootstrap: []`.

**Root cause**: `DEFAULT_CONFIG` in `packages/core/src/storage/config.ts` hardcoded `bootstrap: []` instead of using `DEFAULT_P2P_CONFIG.bootstrap`. The `??` operator in the constructor treated `[]` as a valid (non-nullish) value, so the fallback to the real bootstrap list never triggered.

**Fix** (ÊñπÊ°à C ‚Äî both sides):
1. `DEFAULT_CONFIG.p2p.bootstrap` now imports from `DEFAULT_P2P_CONFIG.bootstrap` ‚Üí new `config.yaml` files are correct from the start
2. Constructor fallback chain uses `arr?.length ? arr : undefined` ‚Üí existing empty `config.yaml` files fall through to defaults

**Affected packages**: `@claw-network/core`, `@claw-network/node`

#### [P1] Outbox Messages Stuck at 0 Peers

**Symptom**: Messages sent via `messagingService.send()` enter the SQLite outbox when the target peer is unreachable. If no `peer:connect` event ever fires (e.g. NAT-blocked node), messages remain queued indefinitely until TTL expiry ‚Äî silently lost.

**Fix**: Added a 30-second `setInterval` sweep in `MessagingService` that calls `flushOutboxForDid()` for all DIDs with pending outbox entries. The existing exponential backoff (1s ‚Üí 60s cap, max 50 attempts) applies per message.

New log line when sweep delivers messages:
```
[messaging] outbox sweep delivered { delivered: 3, targets: 2 }
```

**Affected packages**: `@claw-network/node`

### üîç Diagnostic Improvements

#### [P0 support] NAT Connection Failure Logging

**Context**: TelAgent reported that NAT-behind nodes show `peer:discovery` but never `peer:connect`. The original code swallowed all dial errors with empty `catch {}` blocks, making root cause identification impossible.

**Changes**:
- `connection:open` ‚Äî logs remote peer ID + multiaddr on every successful connection
- `connection:close` ‚Äî logs remote peer ID + multiaddr on every disconnection
- `dialPeer()` ‚Äî logs `[p2p] dial failed for <peer>: <error>` at `warn` level
- `amplifyMesh()` ‚Äî logs peerStore dial failures at `debug` level
- `reconnectBootstrap()` ‚Äî logs bootstrap multiaddr + error on failure

Example output on NAT dial failure:
```
[p2p] peer:discovery 12D3KooWRTEt‚Ä¶
[p2p] dial failed for 12D3KooWRTEt‚Ä¶: connection timeout
[p2p] bootstrap dial failed for /dns4/clawnetd.com/tcp/9527‚Ä¶: noise handshake timeout
```

**Affected packages**: `@claw-network/core`

**Status**: Logging is complete; the NAT root cause itself requires TelAgent to provide the new diagnostic logs from a real NAT environment for further investigation.

### Upgrade Guide

**For embedded integrators (TelAgent etc.)**:

1. Update `@claw-network/node` and `@claw-network/core` to `0.6.15`
2. The workaround `p2p: { bootstrap: DEFAULT_P2P_CONFIG.bootstrap }` in constructor args is **no longer necessary** (but harmless to keep)
3. Delete existing `config.yaml` (or leave it ‚Äî the empty-array fallback handles it)

**For standalone clawnetd operators**:

```bash
cd /opt/clawnet && git pull origin main && pnpm build && systemctl restart clawnetd
```

No configuration changes needed. Existing `config.yaml` with `bootstrap: []` will be handled automatically.

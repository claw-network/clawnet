## v0.6.14 — Security Hardening + Auth Hotfix

This release combines the **v0.6.13 security hardening** (4 High + 2 Medium findings from a security audit) with a **critical hotfix** for an auth deadlock regression that blocked fresh node bootstrapping in v0.6.13. If upgrading from v0.6.12 or earlier, this is the recommended target version.

### Highlights

- **6 security audit findings resolved** — CORS, auth, CSRF, atomic storage, snapshot integrity, rate limiting
- **Auth deadlock hotfix** — Fresh zero-key nodes can now create their first API key via `POST /api/v1/admin/api-keys` from localhost (v0.6.13 blocked this with 401)
- **`ClawNetNode.createApiKey()` public API** — Embedding applications can bootstrap auth programmatically without HTTP or private property access
- **Atomic event storage** — `EventStore.appendEvent()` now uses LevelDB `WriteBatch` with in-process mutex
- **Crash-safe snapshots** — Atomic write-then-rename for all snapshot files
- **Dockerfile OCI labels** — GHCR container images now auto-link to the GitHub repository page

---

### ⚠️ Breaking Changes

1. **CORS default changed from `*` to deny-all** — Non-mainnet nodes previously allowed all cross-origin requests. Operators who need browser-based API access must now explicitly set `corsOrigins`.

2. **Zero-key nodes now return 401 on all networks** — Previously, fresh testnet/devnet nodes allowed unauthenticated access. Now all networks enforce authentication. Create an API key from localhost first (see [Migration Guide](#-migration-guide)).

3. **Rate limiter requires `trustedProxies` config** — `X-Forwarded-For` is no longer trusted by default. If behind a reverse proxy, set `trustedProxies: ["127.0.0.1"]`.

### 🐛 Bug Fixes

#### [P0] Auth Deadlock on Zero-Key Nodes (v0.6.13 regression)

**Symptom**: `curl POST /api/v1/admin/api-keys` on a fresh node returns 401 instead of 200. All SDK/HTTP integrators unable to initialize nodes.

**Root cause**: The v0.6.13 zero-key auth hardening applied 401 rejection before requests reached the admin router. The admin route's own `isLocalhost()` guard never had a chance to execute — creating a deadlock where you need an API key to create an API key.

**Fix**: Added `/api/v1/admin/` to the `PUBLIC_ROUTES` whitelist in the auth middleware. Admin routes are already protected by:
1. `isLocalhost(req)` — only 127.0.0.1 / ::1 / ::ffff:127.0.0.1
2. `isCsrfSafe(req)` — blocks cross-origin browser POSTs

This fix does **not** reduce the security posture introduced in v0.6.13:

| Security property | v0.6.13 | v0.6.14 |
|-------------------|---------|---------|
| Non-localhost admin requests | Auth middleware 401 | Admin route `isLocalhost()` 403 |
| Cross-origin browser CSRF | CORS policy blocks | CORS + `isCsrfSafe()` double block |
| Zero-key non-admin routes | 401 | 401 (unchanged) |
| Key validation for keyed nodes | Normal | Normal (unchanged) |

### 🔒 Security Fixes (from v0.6.13)

#### [High] CORS + Auth: Localhost-CSRF Takeover (Finding #1)
- CORS default changed from `["*"]` to `[]` (deny all cross-origin) for all networks.
- Operators must explicitly configure `corsOrigins` for browser access.

#### [High] Auth: Unauthenticated Access on Fresh Nodes (Finding #1 cont.)
- Zero-key nodes now return `401 Unauthorized` on all networks (HTTP and WebSocket), not just mainnet.
- WebSocket upgrade handlers (`ws-messaging`, `ws-delivery-stream`) share a `requireWsAuth()` helper that enforces the same policy as HTTP middleware.

#### [High] Admin API: CSRF via Cross-Origin Requests (Finding #1 cont.)
- Admin endpoints check the `Origin` header. If present and not a localhost URL, the request is rejected with `403 Forbidden`.
- Browsers always send `Origin` on cross-origin POSTs, blocking CSRF from malicious pages while allowing `curl`/SDK calls.

#### [High] EventStore: Non-Atomic Append (Finding #2)
- Added `batch()` method to the `KVStore` interface — optional, with atomic semantics when supported.
- `LevelStore` implements `batch()` via LevelDB native `WriteBatch` (atomic and crash-safe).
- `MemoryStore` implements `batch()` for test compatibility.
- `EventStore.appendEvent()` now uses a single `batch()` call for the sequence entry, hash→seq index, and counter bump — replacing three sequential `put()` calls.
- Added an in-process mutex to prevent concurrent callers from reading the same sequence number.

#### [High] Snapshots: Metadata File Pollution (Finding #3)
- `findLatestSnapshotFile()` now explicitly excludes `latest.meta.json` and `.tmp` files from the fallback directory scan.
- `saveSnapshot()` uses write-to-temp-then-`rename()` for all three file writes. `rename()` is atomic on POSIX, preventing partial/corrupt files on crash.

#### [High] WebSocket: Auth Bypass on Zero-Key Nodes (Finding #4)
- Both `ws-messaging` and `ws-delivery-stream` upgrade handlers reject connections with `401 Unauthorized` when no API keys are configured.
- Extracted shared `requireWsAuth()` helper to keep WebSocket auth in sync with HTTP.

#### [Medium] Contracts: Read/Write Inconsistency (Finding #5)
- Contract GET endpoints now prefer `ContractsService` (chain/indexer backend) when available, falling back to the event-sourced `contractStore` only when no chain backend is configured.
- Chain-backed responses normalized to include an `id` field alongside `contractId` for SDK compatibility.

#### [Medium] Console: Brute-Force + Spoofable Rate Limit (Finding #6)
- `getClientIp()` now only trusts `X-Forwarded-For` when `req.socket.remoteAddress` matches a configured `trustedProxies` list (default: `[]`).
- Added a dedicated per-IP rate limiter on `/api/v1/auth/verify-passphrase`: max 5 attempts/min, with escalating lockout (1 → 5 → 15 min) after 10 consecutive failures.

### ✨ Features

#### `ClawNetNode.createApiKey()` Public Method

A stable API for embedding applications (e.g. TelAgent) to bootstrap auth without HTTP or private property access:

```typescript
import { ClawNetNode } from '@claw-network/node';

const node = new ClawNetNode({ dataDir: '/data/clawnet' });
await node.start();

const result = node.createApiKey('my-agent');
if (result) {
  console.log('Key:', result.key);   // 64-char hex, returned only once
  console.log('ID:', result.id);
  console.log('Label:', result.label);
}
```

Replaces the previous workaround:
```typescript
// ❌ Before: accessing private property, may break on upgrade
const store = (node as any).apiKeyStore;
const record = store.create(label);

// ✅ After: stable public API
const result = node.createApiKey(label);
```

### 📋 Migration Guide

#### 1. Create an API key (required for fresh nodes)

```bash
curl -X POST http://127.0.0.1:9528/api/v1/admin/api-keys \
  -H "Content-Type: application/json" \
  -d '{"label": "my-key"}'
# Returns 200 + the API key (shown only once)
```

Or programmatically for embedded nodes:
```typescript
const result = node.createApiKey('my-agent');
```

#### 2. Configure CORS (if using browser-based access)

```json
{ "corsOrigins": ["https://your-frontend.example.com"] }
```

#### 3. Configure trusted proxies (if behind Caddy/nginx)

```json
{ "rateLimit": { "trustedProxies": ["127.0.0.1"] } }
```
